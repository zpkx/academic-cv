### Playwright Test Automation  `2025 — 2026`

> **技术栈**：Playwright · TypeScript · Page Object Model · monocart-reporter · oracledb · TA JWT (RSA) · DUO OIDC · OAuth2 JWT · Express · Multer · Docker · Jenkins · Conjur · Node.js

**📌 背景与动机**

团队多个项目（RCE、CART、Supply Chain Finance Tools 等）长期缺少 E2E 自动化测试，质量保障高度依赖手工回归，导致发布节奏受限、回归成本高。发起此 Initiative，目标是为团队引入一套**现代化、可复用、跨项目通用**的端到端自动化测试体系。

**🎯 价值与产出**

- **跨项目测试框架**：基于 **Playwright + TypeScript + Page Object Model** 从 0 搭建跨应用通用的自动化测试框架，覆盖 **CART（E2E + API）、RCE（E2E / Regression / Smoke）** 多套测试矩阵，并沉淀 **AG Grid 自动化工具、Oracle DB 直连数据校验、邮件通知** 等通用能力；
- **企业级认证适配**：设计并实现 **TA JWT（RSA 签名）测试态认证方案**，与生产应用配套联动（私钥经 **Conjur** 托管、CI 中安全注入），彻底解决企业 SSO 在自动化场景下无法登录的难题，让测试稳定无人值守运行；
- **CI/CD 集成与容器化执行**：**自构建并维护专用 Playwright Docker 镜像**（含 Oracle Instant Client 等依赖）保证环境一致性，通过 **Jenkins Pipeline + 矩阵参数（应用 / 环境 / Project / Tag）** 调度执行，每次 PR / 发布自动运行回归套件，显著降低回归成本、加快发布节奏；
- **企业级测试报告服务**：自建 **Test Report Server**（Express + Multer），**DUO OIDC** 保护报告浏览、**OAuth2 JWT** 保护上传接口，**Docker** 化部署，集中托管历史报告，便于团队追踪趋势与定位回归问题；
- **跨项目复用与标准化**：作为可复用模板被团队多个项目采纳，统一了团队的 E2E 测试标准与最佳实践。

### Video Downloader `2025/6 — 2025/7`

> **技术栈**：Python 3.12 · yt-dlp · PyYAML · Docker · Docker Compose · GitHub Actions · GHCR · Bash
> **开源仓库**：[github.com/zpkx/video-downloader](https://github.com/zpkx/video-downloader)（个人 GitHub 公开）

**📌 背景与动机**

个人业余项目，**源码已在个人 GitHub 开源公开**，解决多平台视频批量下载与归档的痛点：手工维护 URL 列表、按类别分目录保存、避免触发限流，并希望在 NAS / 容器环境中**改配置即自动下载**，无需反复手工执行命令。

**🎯 价值与产出**

- **核心下载引擎**：基于 **yt-dlp** 封装 `VideoDownloader` 类，支持单 URL、多 URL、按类别批量下载，以及 **info-only / dry-run** 等非破坏性操作模式；
- **YAML 驱动配置**：用 **urls.yaml** 定义分类目录、画质、延迟区间与 URL 列表，用 **config.yaml** 集中管理 yt-dlp 选项（字幕、合并格式、重试策略等），支持 `config/` 目录自动探测加载；
- **Docker 化与自动化**：提供 **Dockerfile + docker-compose**（watcher / oneshot / manual 多服务模式），实现 **urls.yaml 变更即触发下载** 的文件监控；支持可配置 **UID/GID** 避免挂载卷权限问题；
- **运维与发布**：配套 `docker-run.sh` 等辅助脚本、分级日志（`logs/video_download.log`）、Cookie 自动探测；通过 **GitHub Actions** 构建并发布镜像至 **GHCR**（`ghcr.io/zpkx/video-downloader`），便于家庭 NAS 长期拉取运行。
